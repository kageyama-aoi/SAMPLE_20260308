# SPEC - 技術仕様・要件定義

## 機能要件

### 1. テキスト入力・解析
- フリーテキスト（代行依頼メッセージ）を貼り付けるテキストエリアを提供する
- 貼り付け後、「解析」ボタンで以下の情報をルールベース（正規表現）で自動抽出する：
  - 日付（例：5/17(日)、3月21日（土）など複数フォーマットに対応）
  - ジム名（例：エスフォルタ八王子、NAS永山など）
  - 時間（開始〜終了、例：9:45〜10:45）
  - クラス名（例：マーシャルワークアウト、ストライクボクシング45）
  - 調整さんURL（chouseisan.com のURL）
  - 依頼者名（テキスト内に含まれていれば抽出、なければ空欄）
- 解析結果はプレビュー表示し、ユーザーが確認・修正してから登録できる

### 2. 代行一覧表示
- 登録された代行依頼を表形式で一覧表示する
- 表の列：
  | 列名 | 内容 |
  |------|------|
  | 日付 | 代行日 |
  | ジム名 | 施設名 |
  | 時間 | 開始〜終了 |
  | クラス名 | プログラム名 |
  | 依頼者 | 依頼した人の名前 |
  | 代行者 | 引き受けた人の名前 |
  | ステータス | 未決 / 対応済み |
  | URL | 調整さんリンク |

### 3. ステータス管理
- 各行のステータスを「未決」「対応済み」で切り替えられる
- 代行者名を入力したタイミングで「対応済み」に自動変更する（任意）

### 4. 編集・削除
- 各行を個別に編集・削除できる

### 5. AIフォールバック機能
- 解析結果が0件、または解析精度が低いと判断されたとき、フォールバックUIを表示する
- フォールバックUIには以下を含む：
  - 「うまく解析できませんでした」などのメッセージ
  - 「AIに解析してもらう」ボタン
  - ボタン押下で、元テキスト＋抽出指示文（JSON形式で出力するよう促すプロンプト）をクリップボードにコピーする
  - 「コピーしました！ChatGPTやClaudeに貼り付けてください」と案内する
- プロンプトのフォーマット：
  - 元テキストをそのまま含める
  - 「以下のテキストから代行依頼情報を抽出し、JSON形式（date, gym, time, className, requester, url）で出力してください」という指示文をセットにする

### 6. データ保存
- データはブラウザのローカルストレージに保存する
- ページをリロードしても데이터が保持される

## 非機能要件

- ログイン・認証機能なし（シングルユーザー前提）
- スマホ・PC両対応（レスポンシブデザイン）
- サーバー不要（静的ファイルのみ、ブラウザで完結）
- 外部APIへの依存なし（テキスト解析はクライアントサイドのルールベース処理）

## 技術構成

- **フロントエンド**：HTML / CSS / JavaScript（バニラJS）
  - フレームワークなし（シンプルに静的ファイルで完結）
  - レスポンシブ対応（CSS Flexbox / Grid）
- **データ保存**：localStorage
- **テキスト解析**：正規表現ベースのパーサー（クライアントサイド）
- **ホスティング**：GitHub Pages などで静的配信（任意）

## 対応する入力フォーマット例

### フォーマット1（インライン形式）
```
5/17(日) エスフォルタ八王子　9:45〜10:45マーシャルワークアウト
5/17(日) NAS永山　14:30〜15:15 マーシャルワークアウト
https://chouseisan.com/s?h=xxxxxxxx
```

### フォーマット2（箇条書き形式）
```
3月21日（土）
・場所：品川健康センター
・時間：18:30〜19:15
・プログラム：ストライクボクシング45
```

## 画面構成

1. **ヘッダー**：アプリタイトル
2. **入力エリア**：テキストエリア + 解析ボタン
3. **プレビューエリア**：解析結果の確認・修正フォーム + 登録ボタン
4. **一覧テーブル**：登録済み代行依頼の一覧（ステータスフィルターあり）

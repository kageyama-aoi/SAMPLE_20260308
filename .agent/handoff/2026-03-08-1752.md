## 試したこと・結果

### 成功したアプローチ
- `id` なしJSONのインポートバグ：`.map(en => ({ ...en, id: en.id || generateId() }))` で事前にidを付与してからフィルターをかける方式で解決
- AIフォールバックを「解析失敗時のみ表示」から「常時アクセス可能なボタン」に変更し、ユーザーが任意のタイミングで使えるようにした
- `navigator.clipboard.writeText` + `legacyCopy()` のフォールバックでクリップボードコピーを堅牢化

### 失敗・修正したアプローチ
- 当初のインポートコードは `en.id` が truthy なエントリのみ通すフィルターだったため、AIが出力した id なしJSONが全件スキップされた（0件インポート）→ id生成を先行させる方式に変更

## 注意点・ブロッカー
- ファイル構成：`index.html`（ルート）、`src/js/app.js`、`src/css/style.css`
- ルートの旧 `app.js` `style.css` は削除済み
- Google Material Icons はCDN経由のため、オフライン環境ではアイコンが表示されない
- エクスポートしたJSONには `id` フィールドが含まれるが、AIが生成したJSONには含まれない場合がある → インポート時に自動生成する実装済み
- 重複インポートの検出は `id` ベース。同一データを id なしで複数回インポートすると重複登録される（許容範囲）
